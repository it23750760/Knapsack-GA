2) GA design (10 min)

Binary encoding: chromosome is a length-n bitstring where 1 = take item. Fitness (choose one):

Hard penalty

fitness(x) = value(x) − M · max(0, weight(x) − C)

Use large M (e.g., 10×max(v)). Enforces feasibility strongly.

Soft penalty

fitness(x) = value(x) / (1 + max(0, weight(x) − C))

Keeps overweight individuals in play (helps roulette/diversity).

Operators (pick sets to compare)

  • Selection: tournament(k), roulette, rank, truncation.
  • Crossover: 1-point, 2-point, uniform (rate pc).
  • Mutation: bit-flip (rate pm per gene).
  • Elitism: copy top E to next generation.

def make_fitness(v, w, C, hard=True, M=1000):
    def fitness(ind):
        val = sum(g*vi for g,vi in zip(ind, v))
        wt  = sum(g*wi for g,wi in zip(ind, w))
        of  = max(0, wt - C)
        return (val - M*of,) if hard else (val / (1+of),)
    return fitness

def build_toolbox(n, fitness_fn, pc=0.9, pm=0.02, tourn_k=3):
    creator.create("FitnessMax", base.Fitness, weights=(1.0,))
    creator.create("Individual", list, fitness=creator.FitnessMax)

    tb = base.Toolbox()
    tb.register("attr_bool", random.randint, 0, 1)
    tb.register("individual", tools.initRepeat, creator.Individual, tb.attr_bool, n=n)
    tb.register("population", tools.initRepeat, list, tb.individual)
    tb.register("evaluate", fitness_fn)
    tb.register("mate", tools.cxTwoPoint)
    tb.register("mutate", tools.mutFlipBit, indpb=pm)
    tb.register("select", tools.selTournament, tournsize=tourn_k)
    return tb

def run(v, w, C, gens=200, pop_size=150, **ops):
    tb = build_toolbox(len(v), make_fitness(v,w,C, hard=True, M=10*max(v)), **ops)
    pop = tb.population(n=pop_size)
    hof = tools.HallOfFame(1)
    stats = tools.Statistics(lambda ind: ind.fitness.values[0])
    stats.register("avg", lambda s: sum(s)/len(s))
    stats.register("max", max)
    algorithms.eaSimple(pop, tb, cxpb=ops.get("pc",0.9), mutpb=ops.get("pm",0.02),
                        ngen=gens, stats=stats, halloffame=hof, verbose=False)
    return hof[0], hof[0].fitness.values[0]

3C) pymoo scaffold

# pymoo_ga.py
import numpy as np
from pymoo.core.problem import Problem
from pymoo.algorithms.soo.nonconvex.ga import GA     # :contentReference[oaicite:11]{index=11}
from pymoo.optimize import minimize
from pymoo.core.sampling import Sampling
from pymoo.core.crossover import Crossover
from pymoo.core.mutation import Mutation

class KP(Problem):
    def __init__(self, v, w, C):
        super().__init__(n_var=len(v), n_obj=1, n_constr=0, xl=0, xu=1, type_var=bool)
        self.v, self.w, self.C = np.array(v), np.array(w), C
    def _evaluate(self, X, out, *args, **kwargs):
        W = X @ self.w; V = X @ self.v
        of = np.maximum(0, W - self.C)
        f  = -(V - 1000*of)          # pymoo minimizes → negate
        out["F"] = f.reshape((-1,1))

problem = KP(v, w, C)
algorithm = GA(pop_size=150)         # customize operators if desired
res = minimize(problem, algorithm, ("n_gen", 200), seed=42, verbose=False)
best_x = res.X; best_fit = -res.F[0]
Tip: fix random seeds when comparing operators (DEAP: random.seed(…), PyGAD: random_seed in GA, pymoo: argument seed).

4) Experiments (45–55 min)

A. Baseline (10–15 min)

  • Start with: tournament k=3, 2-point crossover (pc=0.9), bit-flip (pm=0.02), elitism E=2, gens=200, pop=150.
  • Log per-generation best and average fitness; plot if possible.

B. Operator comparison (15–20 min)

Hold everything else constant; change one thing at a time.

  • Selection: tournament k ∈ {2, 4}, roulette, rank.
  • Crossover: 1-point vs 2-point vs uniform.
  • Mutation rate: pm ∈ {0.01, 0.05, 0.1}.

For each variant, record final best fitness and convergence speed (generations to surpass baseline’s median fitness).

C. (Optional) Harder instances (10–15 min)

  • Try a larger OR-Library instance (~250–500 items) or a “hard” Pisinger instance; note differences in convergence. :contentReference[oaicite:12]{index=12}
